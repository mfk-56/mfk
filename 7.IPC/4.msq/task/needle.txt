编写程序  编译出可执行文件 test
需求
	可执行文件运行时可以携带参数选项 -m -t
	参数选项-m 用于指定程序工作模式：发送消息或接收消息
	参数选项-t 用于指定发送或接收的消息类型

示例：
	# ./test -m send -t 2331  //发送消息类型为2331的消息至默认消息队列
	# ./test -t 7821 -m recv  //从默认消息队列接收消息类型为7821的消息
	# ./test                  //错误：没有指定工作模式 程序退出
	# ./test -t 1222          //错误：没有指定工作模式 程序退出
	# ./test -m recv          //错误：没有指定消息类型 程序退出
	# ./test -t 4771 -m srnd  //错误：无效的工作模式 程序退出

	参数选项 -m 识别的有效工作模式 字符串"send"或"recv"
	如果没有消息队列 根据程序中给定的键值创建一个消息队列
